How I created my blog using Hugo

Apr. 1, 2021

In the last few years, static site generators have gathered lots of attention with the like of Hugo, Gatsby, and many others. These static site generators are mostly used for informative websites such as blog with features that help user to focus more on writing contents.

The most popular generator is Hugo, which is written in the Go language and claimed to have the fastest speed among all generators. As the result, Hugo was chosen to be the generator for this blog.

Creating your site

Hugo has a built-in command that generates a website project for you; this includes all of the files and directories you need to get started.

hugo new site portfolio

This creates the following files and directories:

portfolio/
    |__ archetypes
    |  |__ default.md
    |__ config.toml
    |__ content
    |__ data
    |__ layouts
    |__ static
    |__ themes
  • archetypes: where you place markdown templates for content; hugo uses the files in the archetype folder as models when it generates new content pages
  • config.toml: holds configuration variables for you site
  • data: holds data files in yaml, json, or toml
  • layouts: where you define your site’s look and feel
  • static: holds your css, javascript file, images, and any other assets
  • themes: holds any themes you download or create

Layouts

hugo requires you to create a layout for the home page, a single page layout for content pages, and a list layout that indexs content pages into categories.

hugo uses a separate layout for the home page because it assumes that you’ll want your home page to have a different look than other content pages. in my case, the homepage contain a brief introduction that lead to the about page, a list of recent articles, a list of my personal projects, and preview of other contents beside technical ones.

in layouts/_default, you can find the single page layout that every content page will use, for e.g., the about page.

when you create a theme, hugo generates the skeletons for parts that doesn’t change such as header, footer, and navigation.

baseof.html will be the base of every other layout you’ll create. instead of including the full skeleton, it’s pulling in other files, orpartials, which contain pieces of layout

the {{ range .Pages }} is where the magic happens. the .Pages collection contains all the pages related to the section you’re working with. the range function lets you iterate over the results so you can display each record in the collection

Content

in order to create new content, for e.g., to create a new “About” page:

hugo new about.md

this will generate the file content/about.md.

Theme

hugo new theme basic

Page Bundles

content organization using page bundles page bundles are a way to group page resources.

Build and Deploy

when you use hugo command without any options, this will generate the pages for your site in a new directory named public, including the html files, xml files for your site’s rss feed, and the sitemap.

you then can upload the contents of the public folder to your web server or cdn.

hugo doesn’t clean up the public folder. you need to delete the generated versions from the public folder.

you can tell hugo to minify the files it generates. this process remove whitespace characters, resulting in smaller file sizes that will make the site download faster.

hugo --cleanDestinationDir --minify
